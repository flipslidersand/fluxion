# ADR-001: ホスト実装言語に Rust を選ぶ

- **日付**: 2026-06-19
- **状態**: Accepted

## 背景

Fluxion のホストランタイム（Wasmtime の呼び出し元、スケジューラー、CLI）を実装する言語を選ぶ必要があった。
候補は Rust と Go。

## 決定

Rust を使う。

## 理由

- `wasmtime` の公式クレートが Rust であり、C API 経由の Go バインディングより型安全かつドキュメントが充実している
- Component Model の bindings generator (`wit-bindgen`) が Rust ファーストで設計されている
- `cargo-component` による Wasm コンポーネントのビルドが Rust と統一できる（ホストとゲストを同じ言語で書ける）
- 所有権・ライフタイムによるメモリ安全性が、ランタイム実装に求められる信頼性と合致する
- 学習目的として「Rust の本格利用」が明示されている

## トレードオフ

- Go に比べてコンパイルエラーが複雑で、学習コストが高い
- 非同期コード（`tokio` + `async/await`）の習熟が必要
- プロトタイピング速度は Go の方が速い場面がある
