# ADR-003: WASI 0.2 から始め、0.3 を後付けする

- **日付**: 2026-06-19
- **状態**: Accepted

## 背景

WASI 0.3 は 2026-06-11 に正式批准されたが、wasmtime のランタイム実装・`cargo-component` のツールチェーン対応はまだ進行中。
WASI 0.3 の主な追加機能は「ネイティブ非同期処理（async streams / futures）」と「キャンセル伝播」。

## 決定

Phase 1〜4 は WASI 0.2（`wasm32-wasip2`）で実装し、Phase 5 以降で WASI 0.3 の非同期 API を段階的に追加する。

## 理由

- WASI 0.2 は wasmtime 20+ で安定動作しており、`cargo-component` のサポートも成熟している
- WASI 0.3 対応ツールチェーンでドキュメント不足・バグに遭遇するリスクが高い
- Fluxion のコアロジック（DAG スケジューラー・Sandbox・永続化）は WASI バージョンに依存しない
- ランタイム呼び出し部分を `fluxion-host` に閉じ込めることで、後から 0.3 API に差し替え可能

## トレードオフ

- WASI 0.3 の非同期ストリームを使ったコンポーネント間パイプラインは Phase 5 以降
- 将来 0.2 → 0.3 の移行コストが発生するが、抽象化層で吸収できる範囲
