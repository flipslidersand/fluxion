# ADR-004: 永続化に SQLite を選ぶ

- **日付**: 2026-06-19
- **状態**: Accepted

## 背景

ワークフロー実行履歴・ジョブ状態・アーティファクトを保存する永続化層が必要。
候補: SQLite / PostgreSQL / sled（組み込み KV）/ ファイル（JSON）。

## 決定

SQLite（`rusqlite` クレート）を使う。

## 理由

- サーバー不要で単一バイナリとして配布できる（CLI ツールとして重要）
- `rusqlite` は Rust 製 SQLite バインディングとして最も成熟している
- SQL で実行履歴をクエリできる（`fluxion status` の実装が容易）
- トランザクションで状態遷移の整合性を保証できる
- アーティファクト（BLOB）も同じ DB に保存できる

## トレードオフ

- 同時書き込みは WAL モードで対応（単一プロセス想定なので許容範囲）
- 分散実行に拡張する場合は PostgreSQL への移行が必要（MVP 外なので考慮しない）
- `sled` より型安全性が低いが、SQL の可読性と実装コストを優先
